## ü§ù Contributing

When adding new documentation:

1. **Choose the right category** - Place docs in the most appropriate folder
2. **Follow naming conventions** - Use kebab-case for filenames
3. **Link from this README** - Add your doc to the relevant section above
4. **Use markdown formatting** - Follow standard GitHub Flavored Markdown
5. **Include examples** - Practical examples help understanding

## üìù Documentation Standards

- **Clear titles** - Use descriptive, concise titles

- **Code examples** - Include runnable code samples where relevant
- **Links** - Use relative links for internal docs
- **Screenshots** - Add visual aids for UI-related features
- **Date stamps** - Include creation/update dates for time-sensitive docs
